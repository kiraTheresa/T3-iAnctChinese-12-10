# 修复可视化页面超出窗口问题

## 问题分析
1. **固定尺寸问题**：DataVisualization组件使用了`width: 100vw`和`height: 100vh`，忽略了外部布局限制
2. **双重高度计算**：主内容区域已设置`min-height: calc(100vh - 60px)`，但可视化组件内部又设置了`height: 100vh`
3. **忽略父容器约束**：可视化组件没有适应父容器的padding和尺寸限制
4. **溢出处理不当**：部分内部元素设置了`overflow: visible`，导致内容超出

## 解决方案
1. **修改DataVisualization.css**：
   - 移除`.data-visualization`类的`width: 100vw`和`height: 100vh`
   - 添加`width: 100%`和`height: 100%`以适应父容器
   - 确保所有内部元素使用`overflow: auto`或`overflow: hidden`

2. **调整布局结构**：
   - 让`.visualization-main`使用`flex: 1`和`min-height: 0`来适应父容器
   - 确保`.visualization-content`和`.visualization-display`也使用弹性布局

3. **移除不必要的样式**：
   - 移除可能导致溢出的固定尺寸
   - 确保所有元素都遵循父容器的约束

## 实施步骤
1. 修改`DataVisualization.css`文件，调整根容器样式
2. 检查并调整内部容器的溢出属性
3. 确保所有容器都使用弹性布局以适应父容器
4. 测试页面在不同尺寸下的表现

## 预期效果
- 可视化页面不再超出窗口
- 正确适应侧边栏展开/收起状态
- 与其他页面保持一致的布局风格
- 在各种屏幕尺寸下都能正常显示
